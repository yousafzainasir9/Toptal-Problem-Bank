Complexity → Technique Lookup

Given constraint n, which techniques are feasible? This lookup helps you eliminate impossible approaches.

Input Size (n) Feasible Complexity Algorithm Class Notes
n ≤ 10 O(n!), O(2^n) Brute force, permutations, subsets Try all combinations. Backtracking acceptable.
n ≤ 20 O(2^n), O(n³) Bitmask DP, subset enumeration 2^20 ≈ 1M. Avoid exponential beyond this.
n ≤ 100 O(n³), O(n² log n) DP, Floyd-Warshall, all-pairs N³ ≈ 1M operations. Feasible in 1s.
n ≤ 500 O(n³), O(n² log n) DP, graph algorithms 500³ ≈ 125M. Tight but usually OK.
n ≤ 10^4 O(n²), O(n² log n) Nested loops, sorting+analysis 10K² = 100M. Must be efficient constant factors.
n ≤ 10^5 O(n log n), O(n√n) Sorting, binary search, segment trees 100K log 100K ≈ 1.7M. Standard online judge limit.
n ≤ 10^6 O(n), O(n log n) Linear pass, hashmap, sorting 1M operations is baseline. O(n) or O(n log n) only.
n ≤ 10^7 O(n) Single pass, prefix sums 10M operations. No nested loops.
n ≤ 10^8 O(n), optimized Cache-friendly, minimal branching 100M. Optimization matters (SIMD, cache locality).
n > 10^8 O(log n), O(1) Binary search, math formula, lookup Only mathematical solutions. No iteration.

Decision Tree

Use this to decide an approach when seeing constraint n:

  • n ≤ 20? Try brute force with backtracking or bitmask DP. O(2^n) is acceptable.
  • n ≤ 500? O(n³) DP is acceptable. Try NLP solutions (edit distance, LCS).
  • n ≤ 10^4? O(n²) nested loops might be tight. Binary search, two pointers, sorting preferred.
  • n ≤ 10^5? Must be O(n log n) or O(n√n). Sorting + analysis, segment trees, Dijkstra.
  • n ≤ 10^6? Only O(n) or O(n log n) solutions. Single pass, hashmap, sort once then analyze.
  • n > 10^6? Must be O(n) single pass or O(log n) lookup. No nested loops, no sorting.
⚠️
Common mistake: Writing O(n²) solution for n=10^5. Test locally with n=10^5 and measure time. If it takes >1s, optimize. Most online judges allow ~1-2s per test.

Edge Cases Master Checklist (20+ Cases)

Before submitting, check EVERY edge case. These catch 80% of WA (Wrong Answer) verdicts.

Input Validation

✓
Empty input: What if array is empty []? String is ""? Answer should handle gracefully (return 0, empty list, false, etc.).
✓
Single element: n=1. Array [5], string "a", tree with one node. Does your solution work?
✓
All identical elements: [5,5,5,5,5]. Does your algorithm fail if all elements are the same?
✓
Duplicates: [1,1,2,2,2,3]. Does your solution assume unique elements?

Ordering

✓
Already sorted (ascending): [1,2,3,4,5]. Does binary search still work? Does merge fail?
✓
Reverse sorted (descending): [5,4,3,2,1]. Common edge case for many algorithms.
✓
Almost sorted (1-2 elements out of place): [1,2,5,3,4]. Does your sort still work?

Numeric Edge Cases

✓
All negative: [-5,-3,-1]. No positive numbers at all. Does min/max fail?
✓
Mix of positive and negative: [-2,0,2]. Watch for overflow when summing.
✓
Zero: Does [0] or [0,0,0] cause division by zero? Off-by-one errors?
✓
int.MaxValue overflow: Adding two large ints can overflow. Use long for sums.
✓
Boundary values: Input at exactly the limits. n=1, n=10^6, price=0, price=10^9. Does code handle?

Data Structure Edge Cases

✓
Empty collections: HashMap with 0 entries, ArrayList with 0 elements. Can you call get()?
✓
Null/undefined: Java/C#: null references. Python: None. Does your code check before dereferencing?
✓
Single element in heap/queue: Extract min from heap with 1 element. Still valid?

String & Array Edge Cases

✓
Empty string "": substring("", 0, 0)? charAt(0) on empty string crashes.
✓
Single character "a": Palindrome check? Longest substring? Should return true/1.
✓
Off-by-one in indexing: Is it i < n or i <= n? [0, n-1] is valid range.

Graph & Tree Edge Cases

✓
Single node tree: Root only, no children. Does traversal work?
✓
Disconnected graph: Two separate components. Does DFS find all nodes?
✓
Cycle in graph: Will DFS revisit nodes? Do you use visited set?
✓
Self-loop: Edge from node to itself (u → u). Does cycle detection handle?

Floating Point & Precision

✓
Floating point precision: 0.1 + 0.2 ≠ 0.3 in binary. Use epsilon for comparison: abs(a - b) < 1e-9.
✓
Division by zero: Avoid a / b when b could be 0. Check first.
✓
Very large/very small numbers: log(0) is undefined. sqrt of negative is NaN.

Comparison & Sorting

✓
Custom comparator edge case: If you define comparator for sorting, test with equal elements. Comparator must handle a==b.
✓
Circular arrays: Indices wrap around. Index n should map to 0. Use modulo: i % n.

Binary Search Template

// Binary Search Template - Find target in sorted array
public int BinarySearch(int[] arr, int target) {
    int left = 0, right = arr.Length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2; // Avoid overflow

        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1; // Search right
        } else {
            right = mid - 1; // Search left
        }
    }

    return -1; // Not found
}

// Variant: Find insertion position (leftmost)
public int SearchInsert(int[] arr, int target) {
    int left = 0, right = arr.Length;

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }

    return left; // Insert position
}
BFS Template (Shortest Path / Level-Order)
csharp 
// BFS Template - Shortest path in unweighted graph
public int BFS(int[][] graph, int start, int end) {
    Queue<int> queue = new Queue<int>();
    HashSet<int> visited = new HashSet<int>();
    Dictionary<int, int> distance = new Dictionary<int, int>();

    queue.Enqueue(start);
    visited.Add(start);
    distance[start] = 0;

    while (queue.Count > 0) {
        int node = queue.Dequeue();

        if (node == end) {
            return distance[end];
        }

        foreach (int neighbor in graph[node]) {
            if (!visited.Contains(neighbor)) {
                visited.Add(neighbor);
                distance[neighbor] = distance[node] + 1;
                queue.Enqueue(neighbor);
            }
        }
    }

    return -1; // No path found
}
DFS Template (Graph Traversal)
csharp 
// DFS Template - Recursive traversal
HashSet<int> visited = new HashSet<int>();

public void DFS(int node, int[][] graph) {
    if (visited.Contains(node)) {
        return;
    }

    visited.Add(node);
    // Process current node

    foreach (int neighbor in graph[node]) {
        if (!visited.Contains(neighbor)) {
            DFS(neighbor, graph);
        }
    }
}

// Iterative DFS using stack
public void DFSIterative(int start, int[][] graph) {
    Stack<int> stack = new Stack<int>();
    stack.Push(start);
    visited.Add(start);

    while (stack.Count > 0) {
        int node = stack.Pop();
        // Process node

        foreach (int neighbor in graph[node]) {
            if (!visited.Contains(neighbor)) {
                visited.Add(neighbor);
                stack.Push(neighbor);
            }
        }
    }
}
Sliding Window Template
csharp 
// Sliding Window Template - Fixed/variable window
// Problem: Find longest substring with at most k distinct chars
public int MaxWindowLength(string s, int k) {
    Dictionary<char, int> charCount = new Dictionary<char, int>();
    int left = 0, maxLen = 0;

    for (int right = 0; right < s.Length; right++) {
        // Expand window: add s[right]
        if (!charCount.ContainsKey(s[right])) {
            charCount[s[right]] = 0;
        }
        charCount[s[right]]++;

        // Shrink window: remove from left while invalid
        while (charCount.Count > k) {
            charCount[s[left]]--;
            if (charCount[s[left]] == 0) {
                charCount.Remove(s[left]);
            }
            left++;
        }

        // Update answer
        maxLen = Math.Max(maxLen, right - left + 1);
    }

    return maxLen;
}
Two Pointers Template
csharp 
// Two Pointers Template - Find pair with target sum
public int[] TwoSum(int[] sorted, int target) {
    int left = 0, right = sorted.Length - 1;

    while (left < right) {
        int sum = sorted[left] + sorted[right];

        if (sum == target) {
            return new int[] { left, right };
        } else if (sum < target) {
            left++; // Need larger sum
        } else {
            right--; // Need smaller sum
        }
    }

    return new int[] { -1, -1 };
}

// Variant: Partition array (e.g., sort colors 0,1,2)
public void Partition(int[] arr, int k) {
    int left = 0, right = arr.Length - 1;

    while (left < right) {
        while (left < right && arr[left] < k) left++;
        while (left < right && arr[right] >= k) right--;

        // Swap
        int temp = arr[left];
        arr[left] = arr[right];
        arr[right] = temp;
    }
}
Union-Find (Disjoint Set) Template
csharp 
// Union-Find with path compression and union by rank
public class UnionFind {
    private int[] parent;
    private int[] rank;

    public UnionFind(int n) {
        parent = new int[n];
        rank = new int[n];

        for (int i = 0; i < n; i++) {
            parent[i] = i;
            rank[i] = 0;
        }
    }

    // Find root with path compression
    public int Find(int x) {
        if (parent[x] != x) {
            parent[x] = Find(parent[x]); // Path compression
        }
        return parent[x];
    }

    // Union by rank
    public bool Union(int x, int y) {
        int rootX = Find(x);
        int rootY = Find(y);

        if (rootX == rootY) return false; // Already connected

        if (rank[rootX] < rank[rootY]) {
            parent[rootX] = rootY;
        } else if (rank[rootX] > rank[rootY]) {
            parent[rootY] = rootX;
        } else {
            parent[rootY] = rootX;
            rank[rootX]++;
        }

        return true; // Successfully merged
    }
}
Prefix Sum Template
csharp 
// Prefix Sum Template - O(1) range queries
public class RangeSumQuery {
    private long[] prefix;

    public RangeSumQuery(int[] nums) {
        prefix = new long[nums.Length + 1];

        // Build prefix sum: prefix[i] = sum of nums[0..i-1]
        for (int i = 0; i < nums.Length; i++) {
            prefix[i + 1] = prefix[i] + nums[i];
        }
    }

    // Query sum of nums[left..right] in O(1)
    public long RangeSum(int left, int right) {
        return prefix[right + 1] - prefix[left];
    }
}

// For 2D matrix: prefix[i][j] = sum of rect from (0,0) to (i-1,j-1)
public long RangeSum2D(long[][] prefix, int r1, int c1, int r2, int c2) {
    return prefix[r2+1][c2+1]
        - prefix[r1][c2+1]
        - prefix[r2+1][c1]
        + prefix[r1][c1];
}
Monotonic Stack Template
csharp 
// Monotonic Stack - Find next greater element
public int[] NextGreaterElement(int[] nums) {
    Stack<int> stack = new Stack<int>();
    Dictionary<int, int> result = new Dictionary<int, int>();

    // Process in reverse to build monotonic decreasing stack
    for (int i = nums.Length - 1; i >= 0; i--) {
        // Pop smaller elements
        while (stack.Count > 0 && stack.Peek() < nums[i]) {
            stack.Pop();
        }

        // Top of stack is next greater (or -1 if empty)
        result[nums[i]] = stack.Count > 0 ? stack.Peek() : -1;

        // Push current element
        stack.Push(nums[i]);
    }

    int[] answer = new int[nums.Length];
    for (int i = 0; i < nums.Length; i++) {
        answer[i] = result[nums[i]];
    }

    return answer;
}
Trie (Prefix Tree) Template
csharp 
// Trie Template - Efficient prefix matching
public class TrieNode {
    public Dictionary<char, TrieNode> Children { get; set; }
    public bool IsEndOfWord { get; set; }

    public TrieNode() {
        Children = new Dictionary<char, TrieNode>();
        IsEndOfWord = false;
    }
}

public class Trie {
    private TrieNode root;

    public Trie() {
        root = new TrieNode();
    }

    // Insert word - O(m) where m = word length
    public void Insert(string word) {
        TrieNode node = root;

        foreach (char ch in word) {
            if (!node.Children.ContainsKey(ch)) {
                node.Children[ch] = new TrieNode();
            }
            node = node.Children[ch];
        }

        node.IsEndOfWord = true;
    }

    // Search for exact word
    public bool Search(string word) {
        TrieNode node = FindNode(word);
        return node != null && node.IsEndOfWord;
    }

    // Search with prefix (no exact match needed)
    public bool StartsWith(string prefix) {
        return FindNode(prefix) != null;
    }

    private TrieNode FindNode(string word) {
        TrieNode node = root;

        foreach (char ch in word) {
            if (!node.Children.ContainsKey(ch)) {
                return null;
            }
            node = node.Children[ch];
        }

        return node;
    }
}
Dijkstra's Algorithm Template
csharp 
// Dijkstra - Shortest path in weighted graph
public long[] Dijkstra(int n, List<(int neighbor, long weight)>[] graph, int start) {
    long[] dist = new long[n];
    for (int i = 0; i < n; i++) dist[i] = long.MaxValue;
    dist[start] = 0;

    PriorityQueue<(long dist, int node)> pq = new();
    pq.Enqueue((0, start), 0);

    while (pq.Count > 0) {
        (long d, int u) = pq.Dequeue();

        if (d > dist[u]) continue; // Skip outdated entry

        foreach ((int v, long w) in graph[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.Enqueue((dist[v], v), (int)dist[v]);
            }
        }
    }

    return dist;
}

Test Day Checklist (Morning Of)

Review this list before you start coding. Takes 5 minutes. Could save 50+ points.

Mental Prep (5 minutes)

  • ✓ Eat a good breakfast. Brain needs fuel for 3-4 hours.
  • ✓ Use the bathroom. Don't code with a full bladder.
  • ✓ Clear your desk. No distractions. No phone.
  • ✓ Close unnecessary tabs. Keep only problem statement, judge, editor.
  • ✓ Take 3 deep breaths. Calm your nervous system. You've trained for this.

Problem-Solving Strategy (Each Problem)

  • ✓ Read carefully: Underline constraints (n ≤ 10^5), requirements (positive integers?), edge cases (empty input?).
  • ✓ Ask clarifying questions: Are duplicates allowed? Is array sorted? Can I modify the input?
  • ✓ Identify pattern: Match against the 35 patterns. Is this sorted array? Two sum? Tree DFS?
  • ✓ Design algorithm: State time and space. O(n log n)? O(n²)? Is it feasible for the constraint?
  • ✓ Write pseudocode: High-level steps before coding. Verify logic works on example.
  • ✓ Code carefully: Use templates. Copy from template, adapt for the problem.
  • ✓ Test on example: Trace through by hand. Does output match expected?

Before Hitting Submit (5 minute review)

  • ✓ Test on PROVIDED example (usually 1-2 test cases shown).
  • ✓ Test on EMPTY input. Array=[], String="", Graph with 1 node.
  • ✓ Test on EDGE case. All duplicates, already sorted, reverse sorted.
  • ✓ Check INDEXING. Array goes [0, n-1]. Not [1, n]. Not [0, n].
  • ✓ Check LOOPS. i < n (not <=). right = arr.Length - 1 (not arr.Length).
  • ✓ Check OVERFLOW. Summing large ints? Use long. mid = left + (right - left) / 2?
  • ✓ Check NULL checks. Do you dereference null pointers? List.Count before accessing [0]?
  • ✓ Check LOGIC. Walk through algorithm once more. Does it handle all cases?
  • ✓ Check NAMING. Variable names make sense? Easy to read?

If You Get Wrong Answer (WA)

  • ✓ Don't panic. WA happens even to experts.
  • ✓ Create a CUSTOM test case that fails. Print intermediate values to debug.
  • ✓ Check the edge cases again: n=0, n=1, all duplicates, negative numbers, zero.
  • ✓ Check indexing: off-by-one is the #1 bug.
  • ✓ Check overflow: if n=10^6 and you sum all, use long not int.
  • ✓ Reread the problem statement. Maybe you misunderstood a requirement.

If You Get Time Limit Exceeded (TLE)

  • ✓ Your algorithm is too slow. Look at the constraint n again.
  • ✓ If n ≤ 10^5, you need O(n log n) or better. O(n²) will TLE.
  • ✓ Check for accidental nested loops. Sorting inside a loop? That's O(n² log n).
  • ✓ Try a more efficient data structure. ArrayList.Contains() is O(n). Use HashSet for O(1).
  • ✓ Memoize if you're recalculating the same subproblems.

Pacing Guide (3-4 hours, 3-5 problems)

  • ✓ Problem 1 (Easy): 20-30 minutes. Should be fast. Warm up.
  • ✓ Problem 2 (Medium): 40-60 minutes. Needs thinking, but doable.
  • ✓ Problem 3 (Medium-Hard): 60-90 minutes. Hard, might need multiple attempts.
  • ✓ Problem 4+ (Hard/Expert): If time permits. Don't spend >90 min on one problem.
  • ✓ Final 5 minutes: Skim all problems. Make sure you haven't missed an easy win.
✓
Pro tip: Read all problems first. Solve the easy ones first to build momentum. Hard problems can wait.

What NOT to Do

  • ✗ Don't code without a plan. Pseudocode first.
  • ✗ Don't submit without testing. At least test on the example.
  • ✗ Don't spend >90 minutes on one problem. Move on, try another.
  • ✗ Don't ignore constraints. If n ≤ 10^5, O(n²) is likely wrong.
  • ✗ Don't assume unique elements if the problem doesn't say so. Handle duplicates.
  • ✗ Don't copy-paste without understanding. Adapt templates to your problem.

Final Tips & Mindset

Common Mistakes Summary

  • Off-by-one errors: i < n, not i <= n. array[i], not array[i+1].
  • Overflow: Sum of large ints overflows. Use long. mid = left + (right - left) / 2 to avoid overflow.
  • Empty edge cases: Empty array, empty string, single element. Always test n=0, n=1.
  • Null pointer dereference: Check if collection is empty before accessing [0].
  • Wrong complexity: O(n²) on n=10^5. Use constraint to validate your approach.
  • Forgot visited set: DFS/BFS on graph? Use visited HashSet to avoid infinite loops.

When You're Stuck

  1. Reread the problem. You might have misunderstood.
  2. Work through an example by hand. Trace the algorithm step-by-step.
  3. Think about edge cases. Empty? Single element? All duplicates?
  4. Ask for a hint. If this is a practice problem, look at the hints or editorial.
  5. Move on. If contest, try another problem. Come back later with fresh eyes.

Build Intuition

Solve 100+ problems and patterns become automatic. When you see "sorted array + target", binary search pops into your head instantly. When you see "shortest path unweighted", BFS is your default. This is pattern recognition. The more you practice, the faster you solve.
